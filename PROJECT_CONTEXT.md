## Regole di lavoro

1. Distinguere sempre tra **Sistema Qualità ufficiale** e **documentazione tecnica della Web App**.
2. Non riscrivere o sostituire parti esistenti senza richiesta esplicita.
3. Non inventare requisiti, norme, soglie o informazioni non presenti nelle fonti ufficiali.
4. Segnalare eventuali incoerenze di versione senza correggerle automaticamente.
5. Prima di modificare codice o struttura, leggere i file già presenti nel repository.
6. Procedere per step: una modifica concreta per volta.
7. Mantenere le risposte brevi, operative e orientate all'azione.
8. Registrare nel repository le decisioni importanti che devono restare valide nelle chat future.

## Obiettivo del progetto

Costruire un sistema digitale semplice, affidabile e progressivo per supportare:

- governance del Sistema Qualità;
- requisiti di accreditamento;
- documenti ed evidenze;
- scadenze, controlli e verifiche;
- miglioramento continuo;
- sviluppo della Web App Janus Quality.

## Istruzione per ogni nuova chat

All'inizio di qualsiasi attività sul repository o sul Sistema Qualità:

1. leggere questo file;
2. consultare il Manuale GOV-QA-001 quando l'attività riguarda governance, procedure o requisiti;
3. verificare lo stato reale dei file prima di proporre modifiche;
4. non ripartire da zero e non riaprire decisioni già definite senza un motivo concreto.
